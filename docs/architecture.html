How Agents Use dbook

Real scenarios where enriched metadata makes the difference

💳
Billing Agent
"Process a refund for order #4821"
dbook provides: payment status enum values (pending, completed, refunded). Agent writes correct WHERE status = 'completed' filter on first try instead of guessing 'paid' or 'success'.
📈
Sales Agent
"Check warehouse inventory for SKU-7392"
dbook provides: FK semantics showing inventory.warehouse_id → warehouses (the storage location). Agent generates correct JOIN path across 3 tables instead of wrong self-join.
📊
Analytics Agent
"Revenue breakdown by customer segment"
dbook provides: auto-detected metrics: SUM(total) GROUP BY segment. Plus example queries showing the exact join from orders through customers to segments. Instant, correct query.
🛠️
Support Agent
"Look up customer jane@example.com"
dbook provides: PII markers on email (sensitivity: high) and phone (sensitivity: medium). Agent knows to redact sensitive fields in responses and limit data exposure.
